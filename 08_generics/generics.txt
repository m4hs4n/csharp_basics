#Generics are templates that allow you to write a single class or method that can work with any data type
(like int, string, or custom classes), while maintaining strict type safety and high performance.

#Think of Generics like a vending machine slot. The slot is engineered to hold and dispense items using the exact same mechanical process,
whether you load it with potato chips, soda cans, or chocolate bars. You don't build a completely separate vending machine for every type of snack;
you build one generic machine that adapts to whatever type you put into it.

#Why do we need them? (The Problem)
Imagine you want to create a utility class that can swap or store two values. Without generics, if you want it to work for integers,
you have to write an integer version. If you want it to work for text, you have to copy-paste the whole code and change the types to strings:

// MESSY: Separate classes for separate types
public class IntBox { public int Data { get; set; } }
public class StringBox { public string Data { get; set; } }

#This causes massive code duplication. (The old alternative was using the generic object type, but that ruins type safety and slows down performance due to boxing/unboxing operations).

#The Generic Solution
With generics, you use a placeholder—conventionally the letter <T> (which stands for Type)—to tell C# that the type will be specified later when the object is actually created.

1. Define the Generic Class (The Template)

// The <T> tells C# this class is a generic template
public class Box<T>
{
    private T _content; // A field of type T

    public void Pack(T item)
    {
        _content = item;
    }

    public T Unpack()
    {
        return _content;
    }
}

2. Use the Generic Class with Different Types
When you use new, you replace <T> with the actual data type you want to store. The compiler dynamically customizes the class for you on the fly:

class Program
{
    static void Main(string[] args)
    {
        // 1. Create a box specifically for integers
        Box<int> numberBox = new Box<int>();
        numberBox.Pack(123); 
        int myNum = numberBox.Unpack(); // No casting needed! Perfectly type-safe.

        // 2. Reuse the EXACT SAME class template for strings
        Box<string> textBox = new Box<string>();
        textBox.Pack("Hello Mahsan");
        string myText = textBox.Unpack(); 
        
        // ❌ The compiler will stop you if you try to put a string in the number box!
        // numberBox.Pack("abc"); -> Compilation Error!
    }
}

#Real-World Example: Where you already see this
You will use generics constantly in professional .NET backend development, especially with collections.
Instead of old-fashioned arrays that have fixed sizes, you will almost always use List<T>.

// A generic list that dynamically resizes and strictly holds your custom objects
List<Employee> staffList = new List<Employee>();
List<int> scoreList = new List<int>();

#Summary Cheat Sheet
- <T> acts as a placeholder: It stands in for a type that will be declared at initialization.

- Write Once, Reuse Anywhere: It eliminates copy-pasting code just to support different primitive data types or objects.

- Type Safety: Unlike using object, generics prevent you from accidentally mixing up types at runtime, catching errors instantly while you type.

- High Performance: It bypasses resource-heavy conversion cycles in memory, making your backend systems run incredibly fast.
